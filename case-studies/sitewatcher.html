Case Study

SiteWatcher

Telegram-native website uptime monitoring with AI hints and optional email alerts. Designed for practical reliability in small teams.

Open Live Demo View Source on GitHub

Problem

Teams need immediate outage visibility, but many monitoring tools are noisy, expensive, or too heavy for small projects.

Constraints

  • Freemium limits required strict resource control
  • Alert fatigue had to be minimized with cooldowns and retries
  • Stack needed to stay simple to deploy on commodity infrastructure

Architecture

  • Backend: FastAPI with async SQLAlchemy and PostgreSQL
  • Check engine: scheduler-triggered async HTTP checks
  • Notifications: Telegram Bot API plus optional Gmail alerts
  • Frontend: React dashboard for sites, logs, and alert settings

Key decisions

  • Used async checks to scale concurrent probes efficiently
  • Added retry and cooldown policy to reduce false-positive noise
  • Stored check history to support trend visibility and debugging
  • Exposed health and status endpoints for transparent operations

Results

  • Fast down/recovered notifications through Telegram delivery
  • Clearer operator confidence from historical check logs
  • Single-project to multi-site path via free/pro plan model

Tradeoffs and next steps

  • Add multi-region checks with region-aware incident aggregation
  • Improve anomaly detection for response-time regressions
  • Expand integrations (Slack, webhooks, and incident workflows)
